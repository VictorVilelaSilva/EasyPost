Usuários e Autenticação

User + Session (NextAuth)
model User {
id        String   @id @default(cuid())
name     String
email    String   @unique
phone    String?
password String
role     Role    @default(PAINTER)
createdAt DateTime @default(now())
budgets Budget[]
}
enum Role { PAINTER  ADMIN }

Produtos / Tintas

Product
model Product {
id            String  @id @default(cuid())
name         String   // "Tinta Coral Padrão"
brand        String   // "Coral"
line         String?  // "premium", "lavável"...
finish       String?  // "acetinado", "fosco"...
packageSize  Float   // tamanho em litros/kg
packageLabel String   // "lata 18L", "galão 3,6L"
yieldM2      Float   // rendimento m² por embalagem
price        Float   // preço unitário R$
isActive     Boolean @default(true)
notes        String?
}

Orçamentos

Budget + BudgetArea + BudgetProduct + BudgetExtraItem
model Budget {
id          String   @id @default(cuid())
painterId  String   // dono do orçamento
// dados do cliente (inline, sem tabela separada no MVP)
clientName   String
clientPhone  String?
clientAddress String?
status     Status  @default(DRAFT)
laborValue  Float   @default(0)
totalMaterials Float @default(0)
totalValue  Float   @default(0)
validity   Int?    // dias de validade
deadline   String?
paymentMethod String?
notes      String?
createdAt  DateTime @default(now())
areas      BudgetArea[]
products   BudgetProduct[]
extraItems BudgetExtraItem[]
}
enum Status { DRAFT SENT APPROVED REJECTED }

model BudgetArea { // ambiente calculado
name      String  // "Sala", "Quarto 1"...
totalArea Float   // m² final pintável
coats     Int     // demãos
areaForPaint Float  // totalArea × coats
notes     String?
}

model BudgetProduct { // tinta usada
productId  String
quantity   Float   // embalagens (arredondado pra cima)
unitPrice  Float
subtotal   Float
}

model BudgetExtraItem { // itens extras
name      String  // "massa corrida", "selador"...
quantity  Float
unitPrice Float
subtotal  Float
}

Decisão: Cliente inline no Budget

No MVP, dados do cliente ficam diretamente no orçamento (sem tabela Client separada). Isso simplifica o schema e o fluxo. Uma tabela Client pode entrar na v2 se o pintor precisar reusar clientes entre orçamentos.
